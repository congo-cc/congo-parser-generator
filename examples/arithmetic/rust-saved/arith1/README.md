# rust-arith1 -- Arithmetic Expression Parser (Parse Only)

A Rust parser for simple arithmetic expressions, generated by
[CongoCC](https://github.com/congo-cc/congo-parser-generator) from the
`Arithmetic1.ccc` grammar.

This is the **parse-only** variant.  It builds an AST from arithmetic
expressions but does **not** evaluate them.  For a fully functional evaluator,
use [rust-arith2](../rust-arith2/) (from `Arithmetic2.ccc`).

## Features

- Parses arithmetic expressions with `+`, `-`, `*`, `/` and parentheses
- Supports integer and decimal numbers (e.g., `42`, `3.14`)
- Correct operator precedence: `*` and `/` bind tighter than `+` and `-`
- Builds an arena-based AST for inspection, traversal, and pretty-printing
- **No evaluation** -- `evaluate()` returns `EvalError::Unsupported`

## Grammar

```
Root                       : AdditiveExpression <EOF>
AdditiveExpression         : MultiplicativeExpression ( (PLUS|MINUS) MultiplicativeExpression )*
MultiplicativeExpression   : (NUMBER|ParentheticalExpression) ( (TIMES|DIVIDE) (NUMBER|ParentheticalExpression) )*
ParentheticalExpression    : OPEN_PAREN AdditiveExpression CLOSE_PAREN
```

## Building

```bash
cargo build
```

## Running the CLI

```bash
# Parse and dump the AST (evaluation is not supported)
cargo run -- "2 + 3 * 4"
```

Output:
```
AST:
AdditiveExpression
  MultiplicativeExpression
    Token(NUMBER, "2")
  Token(PLUS, "+")
  MultiplicativeExpression
    Token(NUMBER, "3")
    Token(TIMES, "*")
    Token(NUMBER, "4")

Evaluation not supported: evaluation is not supported in Arithmetic1; use rust-arith2 instead
Tip: Use arith2-calc (rust-arith2) for expression evaluation.
```

## Running Tests

```bash
cargo test
```

Test suites:
- `tests/parse_files.rs` -- Parses all `.arith` files in `test-data/`
- `tests/evaluate.rs` -- 8 tests covering:
  - `evaluate_root()` returns `EvalError::Unsupported` with helpful message
  - `evaluate()` returns `EvalError::Unsupported` for any node
  - Parsing regression tests (basic, complex, decimal, nested expressions)
  - Parse error detection (empty input, invalid syntax)

## Library API

The Rust crate name is `ex1` (derived from the grammar's Java package name
and configured in `Cargo.toml`).  All public types are re-exported from the
crate root.

### Parsing

```rust
use ex1::Parser;

let ast = Parser::parse("2 + 3 * 4", Some("my-input")).unwrap();
let root = ast.root().unwrap();

// Inspect the AST.
println!("Node count: {}", ast.node_count());
println!("Source text: {}", ast.text(root));
println!("{}", ast.dump(root));
```

### Handling the Unsupported Evaluate

```rust
use ex1::{Parser, EvalError};

let ast = Parser::parse("1+1", Some("test")).unwrap();
match ast.evaluate_root() {
    Ok(value) => println!("Result: {}", value),
    Err(EvalError::Unsupported(msg)) => {
        eprintln!("{}", msg);
        // "evaluation is not supported in Arithmetic1; use rust-arith2 instead"
    }
}
```

## Relationship to rust-arith2

| Feature | rust-arith1 | rust-arith2 |
|---------|-------------|-------------|
| Parsing | Yes | Yes |
| AST construction | Yes | Yes |
| Expression evaluation | No (`Unsupported`) | Yes |
| Grammar source | `Arithmetic1.ccc` | `Arithmetic2.ccc` |

The `Arithmetic2.ccc` grammar extends `Arithmetic1.ccc` with `INJECT` blocks
that define `evaluate()` methods on each AST node type.

## File Structure

```
rust-arith1/
  Cargo.toml             # Crate configuration (lib + bin + tests)
  lib.rs                 # Crate root (generated, re-exports public API)
  main.rs                # CLI binary (hand-written)
  inject.rs              # EvalError + stub evaluate() (hand-written)
  ast.rs                 # Arena-based AST types (generated)
  parser.rs              # Recursive descent parser (generated)
  lexer.rs               # NFA-based lexer (generated)
  tokens.rs              # Token type enum (generated)
  error.rs               # ParseError type (generated)
  visitor.rs             # Visitor + AstMapper traits (generated)
  pretty.rs              # Wadler-Lindig pretty-printer (generated)
  FIXME.md               # INJECT block reference and API guide (generated)
  README.md              # This file (hand-written)
  tests/
    parse_files.rs       # File-based parse tests (generated)
    evaluate.rs          # Evaluation stub tests (hand-written)
  test-data/
    *.arith              # Test expression files
```

Hand-written files are backed up to `../rust-saved/arith1/` for merging after
parser regeneration.
