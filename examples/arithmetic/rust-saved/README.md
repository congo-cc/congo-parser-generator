# rust-saved -- Hand-Written Files for Arithmetic Parsers

This directory contains hand-written Rust files that extend the generated
arithmetic parsers with evaluation logic, CLI binaries, tests, and
documentation.

## Purpose

The files in `examples/arithmetic/rust-arith1/` and
`examples/arithmetic/rust-arith2/` are regenerated by `ant rust-gen` (via
`congocc.jar`).  Regeneration overwrites every `.rs` file in the target
directories, including hand-written additions like `inject.rs` (evaluate
logic), `main.rs` (CLI binary), and the various `tests/*.rs` files.

This `rust-saved/` directory preserves those hand-written files so they
can be merged back after regeneration.

## How Restoration Works

The arithmetic example's
[`build.xml`](../build.xml) `rust-gen` target already automates the
restore step.  After invoking `congocc.jar` to regenerate
`rust-arith1/` and `rust-arith2/` from `Arithmetic1.ccc` /
`Arithmetic2.ccc`, it copies everything in `rust-saved/arith1/` and
`rust-saved/arith2/` back over the regenerated trees with `<copy
todir="..." overwrite="true">`.

So the normal workflow is just:

```bash
cd examples/arithmetic
ant test-rust              # depends on rust-gen, regenerates + restores + runs cargo test
```

If you ever need to do the restore by hand (e.g. after running
`congocc.jar` directly without ant), use:

```bash
cd examples/arithmetic
cp -r rust-saved/arith1/. rust-arith1/
cp -r rust-saved/arith2/. rust-arith2/
cd rust-arith1 && cargo test
cd ../rust-arith2 && cargo test
```

## Editing Workflow

Whenever you change a hand-written file in `rust-arith1/` or
`rust-arith2/`, **also copy the updated file back into the matching
location under `rust-saved/`**.  Otherwise the next `ant rust-gen` will
silently restore the older version and your edits will appear to vanish.

## File Inventory

### arith2/ (evaluator + AST traversal demos)

| File | Description |
|------|-------------|
| `inject.rs` | `EvalError` enum + `Ast::evaluate()` / `Ast::evaluate_root()` implementation |
| `main.rs` | `arith2-calc` CLI binary: parse, evaluate, print result |
| `Cargo.toml` | Adds the `[[bin]]` entry and a `[[test]]` entry for every hand-written test |
| `lib.rs` | Adds `pub use inject::EvalError` re-export |
| `README.md` | Crate-level documentation (build, run, test, API, Visitor / AstMapper / pretty-printer guides) |
| `tests/evaluate.rs` | 41 evaluation tests (positive, negative, edge cases, file-based) |
| `tests/pretty_test.rs` | 3 tests for the Wadler-Lindig pretty-printer |
| `tests/visitor_test.rs` | 3 tests for the read-only `Visitor` trait |
| `tests/astmapper_value_test.rs` | 2 tests for `AstMapper` value modification (`DoubleNumbers`) |
| `tests/astmapper_struct_test.rs` | 2 tests for `AstMapper` structural change with re-parsing |
| `tests/astmapper_struct2_test.rs` | 2 tests for `AstMapper` structural change with **direct evaluation** of the modified AST |

### arith1/ (parse-only)

| File | Description |
|------|-------------|
| `inject.rs` | `EvalError::Unsupported` stub `evaluate()` (mirrors the Java behavior of Arithmetic1.ccc) |
| `main.rs` | `arith1-calc` CLI binary: parse and dump the AST |
| `Cargo.toml` | Adds the `[[bin]]` entry and a `[[test]]` entry for `evaluate.rs` |
| `lib.rs` | Adds `pub use inject::EvalError` re-export |
| `README.md` | Crate-level documentation (build, run, test, parsing API) |
| `tests/evaluate.rs` | 8 tests covering the `Unsupported` stub plus parsing regression checks |
