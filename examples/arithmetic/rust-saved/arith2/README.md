# rust-arith2 -- Arithmetic Expression Evaluator

A Rust parser and evaluator for simple arithmetic expressions, generated by
[CongoCC](https://github.com/congo-cc/congo-parser-generator) from the
`Arithmetic2.ccc` grammar.

## Features

- Parses arithmetic expressions with `+`, `-`, `*`, `/` and parentheses
- Supports integer and decimal numbers (e.g., `42`, `3.14`)
- Correct operator precedence: `*` and `/` bind tighter than `+` and `-`
- Parentheses override precedence: `(2+3)*4 = 20`
- Evaluates parsed expressions to `f64` values
- Comprehensive error reporting for parse errors and evaluation errors
  (division by zero, malformed numbers)

## Grammar

The expression grammar implements standard arithmetic precedence via two
production levels:

```
Root                       : AdditiveExpression <EOF>
AdditiveExpression         : MultiplicativeExpression ( (PLUS|MINUS) MultiplicativeExpression )*
MultiplicativeExpression   : (NUMBER|ParentheticalExpression) ( (TIMES|DIVIDE) (NUMBER|ParentheticalExpression) )*
ParentheticalExpression    : OPEN_PAREN AdditiveExpression CLOSE_PAREN
```

Tokens: `NUMBER` (integer or decimal), `PLUS` (+), `MINUS` (-), `TIMES` (*),
`DIVIDE` (/), `OPEN_PAREN` ((), `CLOSE_PAREN` ()). Whitespace is skipped.

## Building

```bash
cargo build
```

## Running the CLI

```bash
# Basic arithmetic
cargo run -- "2 + 3 * 4"       # Output: 14

# Parentheses override precedence
cargo run -- "(2 + 3) * 4"     # Output: 20

# Decimal numbers
cargo run -- "3.14 * 2.0"      # Output: 6.28

# Complex expression
cargo run -- "((1+2)*(3+4))/(5-(6-7))"  # Output: 3

# Arguments can be separate (spaces join them)
cargo run -- 2 + 3             # Output: 5
```

The CLI prints the AST to stderr and the result to stdout.

## Running Tests

```bash
cargo test
```

Test suites:
- `tests/parse_files.rs` -- Parses all `.arith` files in `test-data/`
- `tests/evaluate.rs` -- 41 tests covering:
  - Single values (integer, decimal, zero)
  - Each operator individually
  - Operator precedence (multiplication/division before addition/subtraction)
  - Parenthesized and deeply nested expressions
  - Chained operations
  - Edge cases (negative results, large numbers, whitespace)
  - Division by zero detection (including sub-expressions)
  - Parse error detection (empty input, malformed syntax, unbalanced parens)
  - Evaluation of all test-data files with known expected values

## Library API

### Parsing and Evaluating

```rust
use ex2::parser::Parser;

// Parse an expression string into an AST.
let ast = Parser::parse("2 + 3 * 4", Some("my-input")).unwrap();

// Evaluate the root expression.
let result = ast.evaluate_root().unwrap();
assert_eq!(result, 14.0);
```

### Error Handling

```rust
use ex2::parser::Parser;
use ex2::inject::EvalError;

let ast = Parser::parse("10 / 0", Some("test")).unwrap();
match ast.evaluate_root() {
    Ok(value) => println!("Result: {}", value),
    Err(EvalError::DivisionByZero { location }) => {
        eprintln!("Division by zero at {}", location);
    }
    Err(e) => eprintln!("Error: {}", e),
}
```

### Evaluating Specific Nodes

```rust
use ex2::parser::Parser;
use ex2::ast::NodeId;

let ast = Parser::parse("1 + 2 * 3", Some("test")).unwrap();
let root = ast.root().unwrap();

// Evaluate any subtree by passing its NodeId.
let result = ast.evaluate(root).unwrap();
assert_eq!(result, 7.0);
```

### AST Inspection

```rust
use ex2::parser::Parser;

let ast = Parser::parse("1 + 2 * 3", Some("test")).unwrap();
let root = ast.root().unwrap();

// Dump the AST tree structure (useful for debugging).
println!("{}", ast.dump(root));
// Output:
//   Root
//     AdditiveExpression
//       MultiplicativeExpression
//         Token(NUMBER, "1")
//       Token(PLUS, "+")
//       MultiplicativeExpression
//         Token(NUMBER, "2")
//         Token(TIMES, "*")
//         Token(NUMBER, "3")

// Query node properties.
println!("Node count: {}", ast.node_count());
println!("Source text: {}", ast.text(root));
```

## Error Types

| Error | Cause |
|-------|-------|
| `ParseError` | Input does not conform to the arithmetic grammar |
| `EvalError::DivisionByZero` | Division by zero in the expression |
| `EvalError::InvalidNumber` | A NUMBER token could not be parsed as f64 |
| `EvalError::UnexpectedToken` | Internal: evaluate called on a non-NUMBER token |
| `EvalError::UnexpectedNode` | Internal: evaluate called on an unknown node kind |
| `EvalError::MissingChild` | Internal: expected child missing from AST |

## Relationship to rust-arith1

This crate (`ex2`, from `Arithmetic2.ccc`) extends the base grammar
(`Arithmetic1.ccc`, `ex1`) with expression evaluation.  The `ex1` crate in
`rust-arith1/` provides parsing only -- its `evaluate()` always returns
`EvalError::Unsupported`.

## File Structure

```
rust-arith2/
  Cargo.toml             # Crate configuration (lib + bin + tests)
  lib.rs                 # Crate root (generated, re-exports public API)
  main.rs                # CLI binary (hand-written)
  inject.rs              # EvalError + evaluate() implementation (hand-written)
  ast.rs                 # Arena-based AST types (generated)
  parser.rs              # Recursive descent parser (generated)
  lexer.rs               # NFA-based lexer (generated)
  tokens.rs              # Token type enum (generated)
  error.rs               # ParseError type (generated)
  visitor.rs             # Visitor trait (generated)
  pretty.rs              # Pretty-printer (generated)
  FIXME.md               # INJECT block reference and API guide (generated)
  README.md              # This file (hand-written)
  tests/
    parse_files.rs       # File-based parse tests (generated)
    evaluate.rs          # Evaluation tests (hand-written)
  test-data/
    basic.arith          # 1+1
    decimals.arith       # 3.14 * 2.0 + 0.5
    nested.arith         # ((1+2)*(3+4))/(5-(6-7))
    operators.arith      # 2 + 3 * 4 - 1 / 2
    parentheses.arith    # (2 + 3) * (4 - 1)
    single_number.arith  # 42
```

Hand-written or AI-written files are backed up to `../rust-saved/arith2/` for merging after
parser regeneration.
